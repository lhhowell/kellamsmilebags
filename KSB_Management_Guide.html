First-Time Setup on a New DeviceDo this once per device/browser
1
Open the QC Dashboard
Go to lhhowell.github.io/kellamsmilebags/KSB_QC_Dashboard.html
You'll see a yellow setup banner at the top.
2
Paste your Apps Script URL → Save & Connect
Paste the URL from the gold box above into the yellow banner field → click Save & Connect. The banner disappears and the dot turns 🟢 green.
3
Repeat for Link Tracker and Pass Manager
Open each page — same yellow banner — same URL — same Save & Connect. All three should show 🟢 green.
4
Done — that browser is connected forever
The URL is saved to that browser's local storage. You never need to paste it again on that device unless you clear your browser data.
💡
Working across 3-4 devices?
Do steps 1-3 once on each device. After that, all devices sync automatically through the spreadsheet — check a box on your school laptop, it's checked on your home computer too.
🎟️
Generating Guest PassesPass Manager · for your eyes only
1
Open Pass Manager → scroll to Generate a Guest Pass
Fill in: Label (who is this for), Max Visits (0 = unlimited), Expires After in days (0 = never).
2
Click Generate Pass
A 6-character gold code appears (e.g. D8C7FQ). Use Copy Code or Share to send it to your visitor.
3
Code is logged instantly
Check the Guest_Passes tab in your spreadsheet — the code appears immediately with all details.
4
You get an email when the code is used
Every time a visitor enters their code, you receive a notification email — code used, visits remaining, expiry date.
5
Revoke any time
In the passes table at the bottom, click Revoke next to any active pass. It deactivates instantly — the visitor sees "pass deactivated" on their next attempt.
🎓
Suggested pass settings by use case
Conference demo (DTAL etc.): Visits: 10 · Days: 3
Administrator review: Visits: 0 (unlimited) · Days: 14
Teacher sharing: Visits: 5 · Days: 30
Test pass: Visits: 3 · Days: 1

Turning On the Access GateAfter May 30/31 — when you're ready
✅
Right now the gate is OFF — site is fully public
The gate code is baked into index.html but dormant. The site works exactly as before. Nothing to do until after May 30/31.
1
Open index.html in GitHub's editor
Go to your kellamsmilebags repo → click index.html → pencil icon to edit.
2
Find these two lines near the top and update them
// BEFORE (gate off):
var GATE_ENABLED = false;
var SCRIPT_URL = '';

// AFTER (gate on):
var GATE_ENABLED = true;
var SCRIPT_URL = 'https://script.google.com/macros/s/AKfycbyq...exec';
3
Commit → hard refresh
Commit the change → Ctrl+Shift+R to hard refresh. Visitors now see the code entry screen instead of the site.
4
Generate passes for anyone who needs access
Go to Pass Manager → generate a code → share it with your visitor. They enter it at the gate. You get an email. It's tracked in the spreadsheet.
🔒
What visitors see when the gate is ON
A dark screen matching the site aesthetic — "Smile Bags." logo, a 6-character code input field, and "Contact Laura Howell to request access" at the bottom. Enter a valid code → site unlocks for that browser session.
